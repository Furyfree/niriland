# Niri Keybindings Reference

This document provides a comprehensive reference for all keyboard shortcuts configured in the niriland setup for the niri window manager.

## About the Mod Key

Throughout this document, `Mod` refers to the **Super** key (also known as the Windows key or Command key). This is the default modifier key for most niri window management operations.

## Configuration Files

Keybindings are defined in the following locations:

- **Main bindings**: [configs/modules/.config/niri/modular/binds.kdl](configs/modules/.config/niri/modular/binds.kdl)
- **DMS bindings**: [configs/base/.config/niri/dms/binds.kdl](configs/base/.config/niri/dms/binds.kdl) (auto-generated by DMS)
- **User overrides**: [configs/base/.config/niri/override.d/binds.kdl](configs/base/.config/niri/override.d/binds.kdl) (user-editable)
- **VSCodium**: [configs/base/.config/VSCodium/User/keybindings.json](configs/base/.config/VSCodium/User/keybindings.json)

## Customization

### Adding Your Own Keybindings

To add custom keybindings without modifying the main configuration files:

1. Edit [configs/base/.config/niri/override.d/binds.kdl](configs/base/.config/niri/override.d/binds.kdl)
2. Add your custom bindings in the `binds { }` block
3. Reload niri configuration

Example:
```kdl
binds {
    Mod+Shift+T { toggle-column-tabbed-display; }
    Mod+Alt+Return { spawn "kitty"; }
}
```

### Special Keys

- **Mod**: Super/Windows key (default)
- **XF86 Keys**: Hardware media keys on keyboards and laptops
- **Grave**: The backtick/tilde key (`)
- **BracketLeft/BracketRight**: The `[` and `]` keys

### Key Modifiers

- **repeat=false**: The action only triggers once when key is held
- **allow-when-locked=true**: The keybinding works even when the screen is locked
- **cooldown-ms**: Minimum time between repeated activations (for wheel scroll)

### Mouse Actions

When using mouse wheel bindings, hold the specified modifier keys (e.g., `Mod`) while scrolling to trigger the action.
